REBASE

git rebase это способ объеденить ветки. Он отличается от merge тем, что не сливает общие изменения в целевую ветку, при этом создав еще один коммит. Rebase сжимает все изменения в один патч, а затем вставляет его в целевую ветку, при этом избавляясь от ненужной ветки

Fetch

git fetch обновляет данные одного из того репозитория (или ветки). в то время как git pull это комбинация git fetch и git merge. Git pull сначала обновляет данные, а потом сливает их с вашим локальным репозиторием

Git workflow

git workflow - это модель ветвления (инет кстати говорит, что она устаревшая). В git workflow создается "я понял" 5 веток: main - ветка на которой проект, видимый пользователям; develop - ветка разработки; feature - ответвляется от develop - в ней создаются новые фичи, которые могут быть слиты с develop; release - ветка, которая создается когда в develop уже готов проект; hotfix - Мистер Руслан сказал, что мне не обязательно описывать ветки, поэтому тут я не буду ничего писать.
